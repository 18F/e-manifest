e-Manifest API Examples

Code examples of using the e-Manifest API.

Using the e-Manifest API consists of two major steps: uploading and signing manifests.

The production base URL is https://e-manifest.18f.gov/api/0.1

Notation note: path variables are expressed as :variable_name. The colon is not part of the URI.

Submit a manifest

curl -i -X POST -H 'Content-Type: application/json' \
  --data @manifest.json \
  https://e-manifest.18f.gov/api/0.1/manifest/submit/:manifest_tracking_number

where the :manifest_tracking_number is from Line 4 of form 8700-22.

Fetch a manifest

To retrieve a previously submitted e-Manifest, you need the e-Manifest ID from
the Submit a manifest example.

curl -i -X GET https://e-manifest.18f.gov/api/0.1/manifest/id/:e_manifest_id

If you do not know the e-Manifest ID, but you do have the Manifest Tracking Number from Line 4 of form 8700-22,
you can fetch the e-Manifest object with the Manifest Tracking Number.

curl -i -X GET https://e-manifest.18f.gov/api/0.1/manifest/:manifest_tracking_number

Update manifest

You may update a previously submitted e-Manifest.

If the manifest has previously been signed, updating it does not change what has been previously signed
(see Sign a manifest example). You must re-sign the updated e-Manifest.

The update request uses the HTTP PATCH method. See JSON Patch and
JSON Pointer for specification details.

Just as in the Fetch a manifest example, you may use either the e-Manifest ID or the
Manifest Tracking Number.

curl -i -X PATCH -H 'Content-Type: application/json-patch+json' \
  --data @manifest-patch.json \
  https://e-manifest.18f.gov/api/0.1/manifest/id/:e_manifest_id

or

curl -i -X PATCH -H 'Content-Type: application/json-patch+json' \
  --data @manifest-patch.json \
  https://e-manifest.18f.gov/api/0.1/manifest/:manifest_tracking_number

An example manifest-patch.json file looks like:

[
   {
      "op" : "replace",
      "path" : "/generator/name",
      "value" : "Generator ZZ"
   },
   {
      "path" : "/generator/mailing_address/zip_code",
      "value" : "11211",
      "op" : "add"
   },
   {
      "op" : "remove",
      "path" : "/manifest_items/1"
   },
   {
      "value" : "no",
      "path" : "/non_hazardous_material_present",
      "op" : "replace"
   },
   {
      "op" : "add",
      "path" : "/transporters",
      "value" : {
         "company_name" : "AAAA Transporter",
         "us_epa_id_number" : "3882",
         "signatory" : {
            "month" : "10",
            "name" : "John Smith",
            "day" : "28",
            "year" : "2015"
         }
      }
   }
]

The response format is the same as in the Fetch a manifest example.

Search for manifests

The e-Manifest API supports full-text search of all e-Manifests. You can search with simple terms, or
by specifying specific fields within which your terms should match. Wildcards, booleans and phrase search
are all supported. See the full query string syntax documentation for more details.

Example of looking for a manifest by Manifest Tracking Number abc123:

curl -i -X GET \
  'https://e-manifest.18f.gov/api/0.1/manifest/search?q=content.generator.manifest_tracking_number:abc123'

You can page through results with the size and from URL query parameters, and sort results by any field.

curl -i -X GET \
  'https://e-manifest.18f.gov/api/0.1/manifest/search?q=abc123&from=0&size=10&sort[]=id:desc'

The search response format looks like:

{
  "total": 99,
  "max_score": 0.123,
  "hits": [
    // list of e-Manifest documents
  ]
}

Authenticate User

You must authenticate with the CDX CROMERR service for signing a manifest. Values in the response are needed to sign the manifest.

curl -i -X POST -H 'Content-Type: application/json' \
  --data @auth-creds.json \
  https://e-manifest.18f.gov/api/0.1/user/authenticate

An example auth-creds.json file looks like:

{
  "user_id" : "CDX user id",
  "password" : "CDX password"
}

The authentication response looks like:

{
  "token": "8f5483a86d3ca682cf7ee8ef514fda1b460b416fd542756645dedea227100719",
  "activity_id": "_633a4568b-811e-471b-92b5-a7a8bca12fea",
  "question": {
    "question_id": "15",
    "question_text": "What was your high school's mascot?"
  },
  "user_id": "CDX user id"
}

You will use the authentication response to Sign your manifest.

Sign manifest

To sign a manifest you must first Authenticate.

You can sign a manifest with either the e-Manifest ID:

curl -i -X POST -H 'Content-Type: application/json' \
  --data @sign-manifest-emanifestid.json \
  https://e-manifest.18f.gov/api/0.1/manifest/sign

where sign-manifest-emanifestid.json looks like:

{
  "token": "8f5483a86d3ca682cf7ee8ef514fda1b460b416fd542756645dedea227100719",
  "activity_id": "_633a4568b-811e-471b-92b5-a7a8bca12fea",
  "id": "your e-Manifest ID",
  "question": {
    "question_id": "15",
    "question_text": "What was your high school's mascot?"
  },
  "user_id": "CDX user id"
}

You may also sign with the Manifest Tracking Number:

curl -i -X POST -H 'Content-Type: application/json' \
  --data @sign-manifest-manifesttrackingnumber.json \
  https://e-manifest.18f.gov/api/0.1/manifest/signByTrackingNumber

where sign-manifest-manifesttrackingnumber.json looks like:

{
  "token": "8f5483a86d3ca682cf7ee8ef514fda1b460b416fd542756645dedea227100719",
  "activity_id": "_633a4568b-811e-471b-92b5-a7a8bca12fea",
  "manifest_tracking_number" : "your Manifest Tracking Number",
  "question": {
    "question_id": "15",
    "question_text": "What was your high school's mascot?"
  },
  "user_id": "CDX user id"
}

Management method codes

Fetch the list of valid hazardous waste report management method codes like this:

curl -i -X GET https://e-manifest.18f.gov/api/0.1/method_code

The response will look like:

[
  {
    "code": "H010",
    "category": "Reclamation and Recovery",
    "description": "Metals recovery including retorting, smelting, chemical, etc."
  },
  // more methods here
]
